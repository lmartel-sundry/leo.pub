H$, pronounced "Hex Money," is a javascript library I built to simplify creating, processing and rendering hexagonal grids in web apps, designed with browser-based games in mind. This was initially part of my Top Secret Video Game (more about that coming soon, hopefully!), but I thought it was cool enough to pull out into a library.

H$ can help you build and display a full game board with just a few lines of code. Create hexagons and attach data and foreground/background image assets, and the H$ grid can render everything for you, manipulate and move the data around and calculate some cool things. Find neighbors, distances, straight lines, etc; there's some animation baked in too.

The screenshot comes from a demo app that shows off a few of the library's features, a GUI tool for generating H$ boards. The coordinate system can be tricky to think about, so this tool allows me to create exactly the board I want and copy-paste it into code. It's sorta fun to play around with, too. Check it out here!

Tech stuff:
  • 100% javascript
  • Rendering done with DOM elements using HTML5 SVG (Scalable Vector Graphics)
  • Uses d3.js for DOM manipulation and animations

Can you never remember quite where you stopped after binging a season and a half of Arrested Development in one night? Want one page that lists the dates and times that all your favorite shows will air their next episode? Then TVTracker is the app for you!

A solo hackathon project that I decided to keep working on, this web app can dynamically find and load data on pretty much any TV show you've ever heard of. You can browse cast lists, airdates and more, as well as track your progress as you watch episodes. There's some snazzy animations involved too. Check it out here!

Tech stuff:
  • Web app built with Ruby on Rails
  • Uses two REST APIs (TVRage and OMDB, if you're curious) to scrape TV show data
  • Animations built with JavaScript and jQuery

I'm building my college a capella group, The Stanford Harmonics, a new website from scratch. In addition to presenting our news, albums, performance dates, etc to the public, the website can be used by group members for scheduling auditions, sharing arrangements and midi files, and editing any of the static content from a web interface. The only reason to change the code is for actual code changes, rather than for content changes. Check it out here!

Tech stuff:
  • Core built with Ruby on Rails
  • Built on top of a Content Management System (Refinery) to allow GUI editing of static webpage content
  • Uses Refinery Engines (small rails apps nested inside the root app) for managing the more complex models

This is a solution I wrote for a recruiting problem involving snake cubes. Given a binary string representing a snake cube (a series of small cubes threaded by a string), this program will tell you whether it's possible to create a single large cube out of it using only legal rotations.

This is a command line java program and I don't have it hosted anywhere, so good luck running it. Honestly, the algorithm is cooler than the results anyway; check out the code here instead.

Tech stuff:
  • Built with Java
  • Basically a recursive algorithm with some error checking and fluff
