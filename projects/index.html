Axis and Allies is a tabletop minatures game I played a lot as a kid. This is my HTML5 tribute to that game: it's animated, multiplayer, completely free, and (I think) a lot of fun, so give it a try! This is (for now) by far my largest solo project.

Tech stuff:
  • Pure javascript (both client and server)
  • Multiplayer over the web, with both synchronous and asynchronous gameplay
  • Heavily database-driven: features autosave and instant replays
  • Infrastructure built with Meteor.js, an awesome (and as of writing half-finished) web framework
  • Built with H$, named "the sexiest hexagonal grid library for sexy people" by TechCrunch*

*well, no

Can you never remember quite where you stopped after binging a season and a half of Arrested Development in one night? Want one page that lists the dates and times that all your favorite shows will air their next episode? Then TVTracker is the app for you!

A solo hackathon project that I decided to keep working on, this web app can dynamically find and load data on pretty much any TV show you've ever heard of. You can browse cast lists, airdates and more, as well as track your progress as you watch episodes. There's some snazzy animations involved too. Check it out here!

Tech stuff:
  • Web app built with Ruby on Rails
  • Uses two REST APIs (TVRage and OMDB, if you're curious) to scrape TV show data
  • Animations built with JavaScript and jQuery

I built my college a capella group, The Stanford Harmonics, a new website from scratch. In addition to presenting our news, albums, performance dates, etc to the public, the website can be used by group members for scheduling auditions, sharing arrangements and midi files, and editing any of the static content from a web interface. The only reason to change the code is for actual code changes, rather than for content changes, which makes it easier to keep everything up to date. Check it out here!

Tech stuff:
  • Core built with Ruby on Rails
  • Built on top of a Content Management System (Refinery) to allow GUI editing of static webpage content
  • Uses Refinery Engines (small rails apps nested inside the root app) for managing the more complex models

H$, pronounced "Hex Money," is a javascript library I built to simplify creating, processing and rendering hexagonal grids in web apps, designed with browser-based games in mind. This was initially part of my Top Secret Video Game (more about that coming soon, hopefully!), but I thought it was cool enough to pull out into a library.

H$ can help you build and display a full game board with just a few lines of code. Create hexagons and attach data and foreground/background image assets, and the H$ grid can render everything for you, manipulate and move the data around and calculate some cool things. Find neighbors, distances, straight lines, etc; there's some animation baked in too.

The screenshot comes from a demo app that shows off a few of the library's features, a GUI tool for generating H$ boards. The coordinate system can be tricky to think about, so this tool allows me to create exactly the board I want and copy-paste it into code. It's sorta fun to play around with, too. Check it out here!

Tech stuff:
  • 100% javascript
  • Rendering done with DOM elements using HTML5 SVG (Scalable Vector Graphics)
  • Uses d3.js for DOM manipulation and animations

This project is a collection of interesting algorithms I've implemented for one reason or another. Various languages. Highlights include:

A* pathfinding algorithm: a JavaScript implementation of A*, often used for finding shortest paths in games. Completely generic and customizable, suitable for plugging into any pathfinding problem. It includes a PriorityQueue implementation and test suite, which could also be used on its own. There's no standalone demo but I use it in this game.

A JavaScript hexagon spiral distance calculator (with a demo): given a pair of hexagons denoted in spiral coordinates (the most inconvenient coordinate system imaginable), this program can find the distance between them in constant time.

A Java (this one's older) snake cube solver: given a binary string representing a snake cube (a series of small cubes threaded by a string), this program will tell you whether it's possible to create a single large cube out of it using only legal rotations. There's no demo because Java, but you can check out the source code or even download and compile it if you really want to.

Tech stuff:

  • These are all licensed under the MIT license--use them if you want them!
  • Each algorithm has a README, check those out for more details
